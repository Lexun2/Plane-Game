# Project Summary: 4 Levels, Bosses, and Scoreboard

Цель: реализовать 4 последовательных уровня с уникальными боссами. Уровни отличаются визуальной темой и механиками противников и падающего оружия. После победы или поражения игрок имеет возможность сохранить счёт, указав имя, и на экране отображается таблица лидеров с датами.

Структура уровней и боссов
- Уровень 1 — Звёздное поле (Space/Stars)
  - Босс: Cosmic Warden
  - Темы и эффекты: космическая среда с звёздами, метеоритные осадки, гравитационные аномалии
  - Выпадающее оружие: плазменные снаряды, метеоритные осколки

- Уровень 2 — Небо и облака (Blue Sky with Clouds)
  - Босс: Azure Skymage
  - Темы и эффекты: голубое небо, облака, воздушные вихри, платформы на высоте
  - Выпадающее оружие: ледяные дротики, облачные лезвия

- Уровень 3 — Пустыня и земля (Desert/Desert Earth)
  - Босс: Sand Sovereign
  - Темы и эффекты: песок, миражи, горячие стени и зыбкие участки
  - Выпадающее оружие: песчаные сабли, бомбы песка

- Уровень 4 — Неоновый стиль фоновой панели (Tron Neon Style)
  - Босс: Neon Overlord
  - Темы и эффекты: неоновый мир с лазерными сетками, футуристическими плитками
  - Выпадающее оружие: неоновые диски, лазерные лучи

Архитектура и форматы данных
- Уровни: assets/levels/level1.json … level4.json
  - В каждом уровне добавлено поле enemySprite, указывающее на общую спрайтовую карту врагов enemys.png (одинаковый файл для всех уровней)
- Боссы: assets/bosses/level1_boss.json … level4_boss.json
- Лидерборд: data/leaderboard.json (модель для локального хранения и возможного бэкенда)
- Визуальные и аудио ассеты хранятся в assets/sprites, assets/audio, assets/backgrounds

Форматы данных (схемы на примере)
- Level JSON: содержит id, name, theme, backgroundAsset, dimensions, boss (ref), weapons, hazards, music
- Boss JSON: id, name, health, attacks, phases, weakness
- Leaderboard JSON: list объектов { name, score, date }

Как будет сохраняться результат
- После окончания раунда игрок может ввести имя; счёт сохраняется в leaderboard.json (локальная загрузка/сохранение, с возможностью синхронизации на сервер при наличии API)
- Отображается топ-лист с датами и именами игроков; обновления происходят по нажатию кнопки "Сохранить" после завершения игры

Ожидаемые риски и ограничения
- Визуальная реализация опасна для производительности в браузере/десктопе; понадобится эффективная отрисовка слоёв, возможно, виртуализация спрайтов на дальних уровнях
- Форматы уровней и боссов требуют согласования с командами художников/аудио для консистентности стилей
- Решение о хранении лидеров локально или на удаленном API зависит от окружения; в первом приближении реализуем локальный JSON
